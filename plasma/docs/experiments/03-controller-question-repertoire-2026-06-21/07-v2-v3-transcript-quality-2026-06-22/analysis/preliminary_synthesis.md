# Transcript-Quality Synthesis

## Summary

The V2/V3 transcript-quality follow-up agrees with the prior final-report-only
judgment for all tested mission classes.

| Mission | Class | Final-report-only winner | Transcript-quality winner |
| --- | --- | --- | --- |
| M1 | narrow-source | V2 | V2 |
| M3 | broad-topic | V3 | V3 |
| M6 | source-conflict | V3 | V3 |

This is stronger than the previous result because it checks whether the
conversation itself improved, not just whether the final report looked better.

## Mission Notes

M1 favors V2. The transcript judge preferred the run that narrowed from storage
and recovery risk into a concrete user trust problem: artifact identity, state,
discovery, download target, and source-read provenance. The winning flow looked
like a one-time reframing followed by recovery.

M3 favors V3. The transcript judge preferred the run that moved through more
product surfaces: source map, ownership, automatic investigation handoff,
UI-less MCP contract, and controller boundary. The repeated shifts helped
because the mission was broad.

M6 favors V3. The transcript judge preferred the run that resisted collapsing
the conflict too early. It reopened the document tension, then recovered into
current C1 default, read-only legacy, experiment-only tooling, and future
investigation boundaries.

## Interpretation

The adaptive-controller signal now has two aligned views:

- final report quality;
- transcript quality.

The result supports a repertoire-selection rule, not one global default. V2
looks safer for narrow-source work. V3 looks stronger when breadth, conflict,
or repeated reframing is part of the mission.

## Limits

The run reused existing clean transcripts and did not add new seeds. The judge
used inconsistent score scales, so numeric scores should not be averaged.
Filesystem path noise remained in some transcript packets, but variant labels
and run IDs were hidden.
